# FAQ

## 为什么 Linux 上测试能过，但真板行为不一定一致？

因为 Linux 主要跑的是 mock HAL，用于验证上层 API 和业务流程，不等同于真实外设时序。

## 当前最适合验证的真板是什么？

`stm32f407_nucleo`。仓库已经默认把它作为 `rtthread` 目标板。

## 烧录后没有串口输出怎么办？

检查以下几点：

1. 是否连接的是 ST-Link 暴露的虚拟串口。
2. 波特率是否为 `115200`。
3. 当前固件控制台使用的是 `USART2`，不是 `USART1`。

## 为什么仓库里不再保留 K210 入口？

因为旧的 K210 路径只有占位代码，没有真实 RT-Thread Nano 产品闭环。当前已经明确删除，避免继续误导构建和文档。
